# 默认假设

当用户只输入菜名时，直接按这些默认值生成，不先追问。

| 项目 | 默认值 | 输出要求 |
| --- | --- | --- |
| 份量 | 2 人份 | 同时给出 1 人和 4 人的缩放提示。 |
| 灶具 | 燃气灶或电磁炉 | 火力同时写设备档位和锅中状态。 |
| 锅具 | 普通炒锅 | 需要时说明铁锅、不粘锅、平底锅差异。 |
| 温度计 | 默认没有 | 温度相关步骤给感官判断，并说明可靠性限制。 |
| 计时器 | 手机计时器 | 超过 3 分钟的等待步骤建议设闹钟。 |
| 称重 | 默认有电子秤 | 同时给无秤体积、个数或手感判断。 |
| 口味 | 正常咸淡 | 盐和糖给基础量与调整范围。 |

默认目标是稳定复现，不追求餐厅级摆盘或复杂技巧。

## 默认输出顺序

1. 完整解释版。
2. 厨房执行版。
3. 已使用的默认假设和可重新适配的信息。

如果用户明确说“快手”“厨房版”“只要步骤”，可以只输出厨房执行版，但不能删除安全提示和失败信号。
