# Memory Merge Rules

记忆合并目标是让菜谱适配用户和家庭成员，同时避免把一次性反馈静默写成长期偏好。

## 优先级

1. 食品安全和过敏、孕期、儿童、疾病相关约束。
2. 用户本次明确输入。
3. 指定家庭成员的忌口和强偏好。
4. 菜谱级已确认偏好和历史反馈建议。
5. 全局用户偏好。
6. Skill 默认值。

## 家庭成员合并

- 用户说“给谁吃”时，只读取这些成员的相关偏好。
- 多人共同进餐时，忌口取并集，口味取保守交集。
- 辣度、盐度、油量有冲突时，默认按更低刺激、更低盐、更低油输出，并给出出锅后分碟补味方案。
- 过敏、孕期、儿童和疾病相关约束必须优先于口感；写入长期记忆前需要确认。

## 历史反馈学习

- 反馈先进入 `feedback_history` 或 `feedback-log`，状态为 `pending-confirmation`。
- Skill 可以提出记忆建议，例如“青椒肉丝下次盐减少 20%”。
- 未经确认的建议只能在当前对话或同一菜谱下作为低置信度提示，输出时标注“建议”，不能静默改长期 profile。
- 记忆建议写入 `memory-candidates.jsonl`，用户确认 candidate 或明确说“以后这道菜都少盐”后，才能写入 `recipe_preferences`。
- 持久化读写通过 `scripts/cooking_memory.py` 完成，真实数据默认在 `~/.rookie-cooking/`。

## 输出要求

- 生成菜谱时说明使用了哪些成员、设备、口味和历史反馈。
- 使用低置信度反馈时必须标注“建议”而不是“默认”。
- 用户可以要求查看、修改、删除或本次忽略任意记忆。
