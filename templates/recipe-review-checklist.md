## 1. 执行完整性

- [ ] 份量、总时间、主动操作时间、难度明确。
- [ ] 原料、工具、设备前置清楚。
- [ ] 每个完整步骤都有操作、时间、火力、目标状态、失败信号、为什么。
- [ ] 厨房执行版每步不超过 3 行。
- [ ] 长时间等待步骤有检查节点或闹钟建议。

## 2. 参数可靠性

- [ ] 主料、盐、酱油、糖、油、水都有精确量或明确区间。
- [ ] 没有无解释的“适量、少许、一会儿、炒熟、差不多”等模糊表达。
- [ ] 感官判断配有可观察状态。
- [ ] 份量缩放说明了非线性参数。
- [ ] 设备差异会影响的时间、火力或锅温已写明。

## 3. 原理匹配

- [ ] 至少引用 1 张原理卡。
- [ ] “为什么”服务当前操作，不写成泛科普。
- [ ] 原理卡和步骤之间没有互相矛盾。

## 4. 安全检查

- [ ] 肉、蛋、海鲜、剩菜、解冻、复热、生熟分开等安全点已覆盖。
- [ ] 安全建议优先于口感建议。
- [ ] 无温度计判断标注可靠性限制。
- [ ] 高风险失败有丢弃或彻底复热建议。

## 5. 厨房友好性

- [ ] 厨房执行版顶部食材清单可作为购物清单。
- [ ] 步骤编号清楚，关键计时点明确。
- [ ] 可离线执行，不依赖持续对话。
- [ ] Review 发现的问题已修正或记录为未解决风险。
